# 2026-05-17 Work Log

## Summary
Added a video pipeline to SAHI object detection, unified the project around a `paths.input_root` / `paths.output_root` config layout, refreshed `starter.sh` to handle the new SAHI mode split, translated the READMEs to English, and dropped two log-analysis helpers under `scripts/`.

## Per-file Changes

### `config.yaml`
- Replaced top-level `input:` / `output:` with a `paths:` section (`input_root`, `output_root`). Each task's `input` is now relative to `input_root`, and outputs are forced into `<output_root>/<task>/{results,logs}/` by the binaries.
- Added `task` ← `sahi_object_detection`, plus `mode: image|video` to drive which SAHI binary runs.
- Tuned defaults: `threshold` 0.50 → 0.80, `overlap_height`/`overlap_width` 0.20 → 0.30.

### `src/common/cli.hpp`
- `BaseConfig` gained `logs_dir` alongside `input` / `output`.
- Added `resolve_task_paths(cfg, root, task_name)`: joins `paths.input_root` with `<task>.input`, builds `<output_root>/<task>/{results,logs}`, and creates the directories.
- Removed `--input` / `--output` CLI options and the legacy top-level `input:` / `output:` YAML keys from `load_base_config()`.
- Each `load_*_config()` now calls `resolve_task_paths()` for its task.
- `load_config()` (classification) now joins `net_path` with `net` to form the full HEF path.

### `src/classification/classifier.cpp`, `src/zero_shot_classification/clip_example.cpp`, `src/SAHI_object_detection/object_detection.cpp`
- Parse config first, then call `logger::set_log_file(cfg.logs_dir.c_str())` so logs land under `<output_root>/<task>/logs/` instead of a hard-coded `logs/`.

### `src/SAHI_object_detection/object_detection_video.cpp` (new)
- New executable for video input. Per-frame stages:
  `VideoCapture.read → slice → preprocess → infer_batch → postprocess(remap + NMM + draw) → VideoWriter.write`.
- Hard-coded `TARGET_FPS = 3.0`; computes `frame_stride = round(in_fps / TARGET_FPS)` so faster sources are sub-sampled and the output file is written at 3 fps.
- `MAX_SLICES = 32` cap; frames exceeding it are skipped.
- EMA-smoothed (`alpha = 0.1`) per-frame latency and FPS drawn in the top-left corner.
- Output file: `<output_dir>/<input_stem>_annotated.mp4` (mp4v).
- Reuses buffers (`preprocessed`, `inputs`, `raw_bufs`, `out_views`) across frames at `MAX_SLICES` size — same pre-allocation pattern as the image binary.

### `src/SAHI_object_detection/CMakeLists.txt`
- Split sources into `COMMON_SOURCES` and built two executables: `SAHI_object_detection` (image) and `SAHI_object_detection_video` (video).

### `src/SAHI_object_detection/build.sh`

## New Output Layout

```
local_data/
├── input/
│   ├── images/
│   └── videos/
└── output/
    ├── classification/{results,logs,reports}/
    ├── zero_shot_classification/{results,logs,reports}/
    └── sahi_object_detection/{results,logs,reports}/
```

- Binaries enforce this via `resolve_task_paths()`; there is no per-task `output:` key.
- Each run writes `logs/YYYY-MM-DD_HHMMSS.csv` (shared columns: `timestamp,level,file,line,func,message`).
- Video runs additionally write `results/<stem>_annotated.mp4`.

## Migration Notes

| Before | After |
|---|---|
| Top-level `input:` / `output:` in `config.yaml` | `paths.input_root` / `paths.output_root` + per-task `input:` |
| `--input` / `--output` CLI flags | Removed; paths derived from config |
| Single SAHI binary `SAHI_object_detection` | Two binaries; `mode` in config selects one |
| `build.sh` (SAHI) ran with hard-coded paths | Requires `--mode image|video`; runs from repo root using `config.yaml` |
| Logs hard-coded to `logs/` | `local_data/output/<task>/logs/` (via `cfg.logs_dir`) |
