SAHI object detection 파이프라인 구현 및 최적화.

## 파일별 수정 내역

### `src/SAHI_object_detection/object_detection.cpp`

- **4단계 파이프라인 구조화**: `run_sahi` 함수 제거, main에서 Phase 1~4 명시적으로 분리
- **`parse_sahi_config` 적용**: `parse_objdet_config` → `parse_sahi_config` 로 교체
- **`frame_bytes` 수정**: `H*W*C*sizeof(float)` 계산 방식 → `model.get_output_frame_size(0)` 으로 교체 (NMS 출력은 shape 기반 계산 불가)
- **`class_count` 수정**: 하드코딩 `COCO_CLASS_COUNT=80` → `output_info.nms_shape.number_of_classes` 로 HEF에서 직접 읽음
- **cross-slice NMM 적용**: 슬라이스별 bbox 수집 후 `apply_nmm()` 으로 중복 박스 병합
- **버퍼 사전 할당**: `preprocessed`, `inputs`, `raw_bufs`, `out_views` 를 이미지 루프 밖 `MAX_SLICES=32` 크기로 선언
- **`VisualizationParams vis` 루프 밖으로 이동**: cfg.threshold는 루프 내에서 변하지 않음
- **`const cv::Mat org`**: ROI 뷰가 살아있는 동안 원본 수정 방지
- **`inv_w` / `inv_h` 역수 계산**: 좌표 변환 나눗셈 → 곱셈으로 대체 (루프 밖에서 한 번 계산)
- **`all_bboxes.reserve(n * 100)`**: push_back 중 reallocation 방지
- **타이밍 로그 추가**: slice / prep / infer / post 각 단계별 ms 측정 및 summary 출력

## 성능 측정 결과 (1920×1080, 슬라이스 8개)

```
model load :  62.6 ms  (최초 1회)
slice      :   0.1 ms  (zero-copy ROI, 거의 무료)
prep       :  18.1 ms  (8장 CPU resize)
infer      :  53.3 ms  (NPU 순차, 6.6 ms/슬라이스)
post       :   8.6 ms  (좌표 리맵 + NMM + 시각화)
total/img  : 224.9 ms
throughput :   3.4 images/s
det        : 96 → 67 boxes (NMM으로 29개 병합)
```

목표 처리량(초당 1장) 대비 **3.4배 여유**.
